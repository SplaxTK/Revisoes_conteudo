Git - Controle de Versão

O Git é um sistema de controle de versão distribuído usado para rastrear alterações no código-fonte durante o desenvolvimento de software. Ele permite que múltiplos desenvolvedores trabalhem no mesmo projeto sem conflitos e mantém um histórico completo de todas as modificações.

Baixar Git

Configuração Inicial

Após a instalação, é essencial configurar suas credenciais para identificar suas contribuições nos repositórios:

Configurar nome de usuário

git config --global user.name "Seu nome"

Define o nome que aparecerá nos commits

Configurar email

git config --global user.email seu@email.com

Associa um email às suas contribuições

Fluxo Básico do Git

O fluxo de trabalho básico no Git envolve estes passos fundamentais:

1

Inicializar repositório

git init

Cria um novo repositório Git na pasta atual

2

Adicionar arquivos

git add nome_do_arquivo

Prepara os arquivos para serem commitados (staging area)

3

Criar uma versão

git commit -m "Mensagem descritiva"

Registra as alterações no histórico do repositório

4

Verificar status

git status

Mostra o estado atual dos arquivos no repositório

5

Ver histórico

git log

Exibe o histórico de commits do repositório

Comandos Básicos do Linux

Conhecer comandos básicos do terminal é essencial para trabalhar com Git e desenvolvimento web. Estes comandos permitem navegar e manipular o sistema de arquivos de forma eficiente.

Navegar entre pastas

cd nome_da_pasta

Entra em uma pasta específica. Use cd .. para voltar um nível.

Listar conteúdo

ls nome_da_pasta

Mostra os arquivos e pastas dentro de um diretório. Use ls -la para ver detalhes.

Criar pasta

mkdir nome_da_pasta

Cria uma nova pasta/diretório. Use mkdir -p pasta/subpasta para criar estrutura aninhada.

Excluir pasta

rm -r nome_da_pasta

Remove uma pasta e todo seu conteúdo. Cuidado: esta ação é irreversível!

Renomear arquivo

mv nome_atual nome_novo

Renomeia ou move um arquivo. Também pode ser usado para mover entre diretórios.

Mover arquivo

mv arquivo caminho_da_pasta/

Move um arquivo para outra pasta. Use caminhos absolutos ou relativos.

Copiar arquivo

cp arquivo destino/

Cria uma cópia do arquivo em outro local. Use cp -r para copiar pastas.

Hospedagem de Sites

A hospedagem web é o serviço que permite que um site seja acessível na internet. Existem diferentes tipos de hospedagem adequados para diferentes necessidades.

Sites Estáticos

Sites construídos apenas com HTML, CSS e JavaScript no lado do cliente. Não possuem processamento no servidor e são servidos exatamente como estão armazenados.

  • Mais rápidos e seguros
  • Fáceis de hospedar e escalar
  • Menos custosos
  • Exemplos: GitHub Pages, Netlify, Vercel

Sites Dinâmicos

Sites que processam informações no servidor antes de enviar ao navegador. Utilizam linguagens como PHP, Python, Node.js e geralmente interagem com bancos de dados.

  • Interatividade avançada
  • Integração com bancos de dados
  • Mais complexos de desenvolver
  • Exemplos: WordPress, aplicações web, e-commerce

Plataformas de Publicação

Existem diversas plataformas para publicar sites na web, cada uma com suas vantagens:

GitHub Pages

Hospedagem gratuita para sites estáticos diretamente de repositórios GitHub. Ideal para portfólios e documentação.

Netlify

Plataforma moderna com deploy contínuo a partir do Git, com recursos avançados como forms, funções serverless e CDN global.

Vercel

Especializada em aplicações frontend com excelente performance, ideal para frameworks como Next.js e React.

Bancos de Dados

Para sites dinâmicos, os dados são armazenados em servidores de banco de dados:

Como funciona

O servidor web se comunica com o banco de dados para buscar, inserir ou modificar informações que são então exibidas ao usuário. Esta comunicação geralmente acontece através de queries SQL ou APIs.

Tipos comuns

  • MySQL - Banco relacional popular para aplicações web
  • PostgreSQL - Banco relacional avançado com mais recursos
  • MongoDB - Banco NoSQL orientado a documentos
  • SQLite - Banco leve embutido em aplicações

Protocolos Web

Os protocolos web são conjuntos de regras que permitem a comunicação entre dispositivos na internet. Dois dos mais importantes são HTTPS e DNS.

HTTPS

HyperText Transfer Protocol Secure - Protocolo de transferência de hipertexto seguro.

Como funciona

O HTTPS adiciona uma camada de segurança ao HTTP através do uso de criptografia SSL/TLS. Isso garante que os dados trocados entre o navegador e o servidor estejam protegidos contra interceptação e modificação.

Para que serve

  • Proteger dados sensíveis como senhas e informações de pagamento
  • Autenticar a identidade do site através de certificados digitais
  • Garantir integridade dos dados durante a transmissão
  • Melhorar SEO (os buscadores priorizam sites HTTPS)

DNS

Domain Name System - Sistema de Nomes de Domínio.

Como funciona

O DNS funciona como uma "agenda de telefones" da internet, traduzindo nomes de domínio legíveis (como www.exemplo.com) em endereços IP numéricos (como 192.168.1.1) que os computadores usam para se conectar.

Para que serve

  • Traduzir nomes de domínio em endereços IP
  • Facilitar a navegação (não precisamos memorizar números)
  • Distribuir o tráfego entre servidores através de load balancing
  • Permitir que sites mudem de hospedagem sem alterar o domínio

Fluxo de uma Requisição Web

  1. Usuário digita um URL no navegador (ex: https://www.exemplo.com)
  2. O navegador consulta o DNS para encontrar o IP do servidor
  3. Estabelece uma conexão HTTPS segura com o servidor
  4. O servidor processa a requisição e envia os arquivos (HTML, CSS, JS)
  5. O navegador renderiza a página para o usuário
